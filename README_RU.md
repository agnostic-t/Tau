# Tau

Проект VPN, построенный на базовых реализациях модулей [Neutrino core](https://github.com/agnostic-t/neutrino-core). Tau поддерживает:

- TCP в качестве транспорта данных по сети
- Null и xOBFS режимы для обфускации трафика
- Plain и xOBFS режимы для рукопожатия
- Null и Yamux режимы для мультиплексации
- Генерация эфемерных ключей раз в N секунд без передачи какой-либо информации между клиентом и сервером (**таймферальные** ключи)

Описание работы алгоритмов находится на https://docs.worldfreeteam.org

## Использование

Настройка и развертывание Tau максимально простые. Все вместе занимает ~5 минут. И на клиенте и на сервере предварительно вам нужно установить `go` (1.26.3)

### Сборка

Установка сводится к скачиванию и компиляции:

```sh
git clone https://github.com/agnostic-t/tau
cd tau

# Для клиента
sudo go build -o /usr/local/bin/taucli ./client/main.go 

# Для сервера
sudo go build -o /usr/local/bin/tauhost ./server/main.go 
```

Убедитесь, что `/usr/local/bin` находится в PATH (или поменяйте путь, куда помещать бинарник)


### Настройка

Настройка также сильно не отличается между клиентом и сервером. Конфигурация сервера:

```json
{
  "bindIP": "0.0.0.0",
  "externalIP": "1.2.3.4",
  "inbounds": {
    "name": {
      "port": 1234,
      "obfs": {},
      "handshake": {},
      "trans": {},
      "mux": {}
    }
  }
}
```

`bindIP` и `externalIP` - IP, куда будет привязываться сервер локально и IP по которому можно подключится к серверу из вне соответственно

Ниже будут приведены подробные настройки параметров инбаундов

Клиент, тут уже по интереснее:

```json
{
  "lproxy": {
    "socks5": "127.0.0.1:9005"
  },
  "tun": {
    "tun": "tunIF0",
    "main": "mainIF0",
    "gateway": "192.168.1.1",
    "enabled": true
  },
  "filter": {
    "direct": "/path/to/direct/list.txt",
    "block": "/path/to/block/list.txt"
  },
  "selcted": "name",
  "servers": {
    "name": {
      "addr": "1.2.3.4:1234",
      "obfs": {},
      "handshake": {},
      "trans": {},
      "mux": {}
    }
  }
}
```

Как можно было заметить у клиента и сервера есть 4 одинаковых поля (`obfs`, `handshake`, `trans` и `mux`). Эти поля соответственно отвечают за то, какие будут протоколы использоваться при: обфускации, хендшейке, транспорте и мультиплексации.

Каждое из 4 полей заполняется следующим образом:

```json
"obfs": {
  "type": "...",
  "settings": {
    "arg": "value"
  }
}
```

Возможно заполнение только `type`, без `settings` вовсе. Сейчас доступны следующие настройки:

1) `obfs`:
    - `type`: `xobfs` или `null`
      - `xobfs` обфусцирует трафик, `null` нет. xobfs требует `settings`
    - `settings`: для xobfs указывается:
      - `psk`: строка, ключ обфускации

2) `handshake`:
    - `type`: `xobfs` или `plain`
      - `xobfs` меняет количество пакетов при подключении, имеет таймферальные ключи, обфусцирует цель рукопожатия. `plain` передает цель подключения как обычный текст
    - `settings`: нужны для `xobfs`:
      - `psk`: строка, ключ обфускации
      - `startJunk`: булево значение, использовать мусорные пакеты или нет
      - `rotateSeconds`: число, раз в сколько секунд менять сигнатуру хендшейка
      - `rotateJunkCount`: булево значение, использовать таймферальные ключи для изменения количества мусорных пакетов или нет
      - `minJunkPacks`: число, минимальное количество мусорных пакетов
      - `maxJunkPacks`: число, максимальное количество мусорных пакетов

3. `trans`:
    - `type`:
      - `tcp`: базовый метод пересылки данных по сети, не требует настроек
      - `http`: транспорт через HTTP-запросы, сервер мимикрирует под сервер Nginx, а клиент под браузер. Для использования требуется настройка:
        - `referer`: валидный URL для доступа к серверу (например `http://127.0.0.1:9001` если сервер VPN доступен по 127.0.0.1 и порт 9001 у этого инбаунда)
        - `keyPath`: путь, который должен использоваться для загрузки данных (например `/api/v1/upload`, может быть любым валидным URI)
        - `userAgent`: нужен только клиенту, используется для подстройки под браузер. Для лучшей безопасности возьмите от Вашего браузера

4. `mux`:
    - `type`: доступны `yamux` и `null`. Первый использует мультиплексирование (1 физическое подключение транспорта на множество запросов), второй не использует мультиплексирование, открывает множество подключений к серверу для передачи информации.

### Запуск

Запуск после сборки и настройки тривиальный:

```sh
taucli /path/to/config.json
tauhost /path/to/config.json
```

> Можно обернуть команду в `nohup ... &` для работы приложения даже при закрытой ssh сессии или закрытом терминале (у клиента)
